## How to Contribute

We welcome contributions of all kinds — documentation improvements, bug fixes, and new features.

### Reporting Issues

If you find a problem or have a suggestion, please [open an issue](https://github.com/arnabnandy1706/documentation/issues/new) on GitHub.

### Submitting Changes

1. **Fork** the repository on GitHub.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/<your-username>/documentation.git
   ```
3. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/my-improvement
   ```
4. **Make your changes** and commit them:
   ```bash
   git add .
   git commit -m "docs: describe your change"
   ```
5. **Push** your branch and open a **Pull Request**:
   ```bash
   git push origin feature/my-improvement
   ```

## Documentation Style Guide

- Use clear, concise language.
- Use [Markdown](https://www.markdownguide.org/) for all documentation.
- Add front matter to every new page.
- Use headings to organize content (`##`, `###`).
- Use code blocks with a language identifier for all code samples.

## Code of Conduct

Please be respectful and constructive in all interactions. We follow the [Contributor Covenant](https://www.contributor-covenant.org/) Code of Conduct.
